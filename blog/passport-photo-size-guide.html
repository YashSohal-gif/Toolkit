Passport & visa photo size requirements

Every country's passport office has slightly different rules — get the pixel size, DPI, and background wrong and your application bounces back. Here's what actually matters.

The four things every photo requirement actually checks

1. Physical size (usually in mm or inches)

Passport photos are specified in physical print dimensions — commonly 2×2 inches (US), 35×45mm (most of Europe, India, UK), or 3.5×4.5cm (parts of Asia). This only matters once the photo is printed, so it's really a combination of pixel dimensions and DPI (see below) — get either wrong and the printed photo comes out the wrong physical size even if it "looks right" on screen.

2. Pixel dimensions and DPI together

A photo's physical print size = pixel dimensions ÷ DPI. So a 2×2 inch photo at 300 DPI needs to be exactly 600×600 pixels. If you upload a 600×600px photo but the file's DPI metadata says 72 instead of 300, some print/upload systems will size it as an 8.3×8.3 inch image instead of 2×2 — technically the same pixels, printed at the wrong size. This is why tools that only resize pixels without also fixing the DPI metadata can produce a photo that fails an automated size check.

3. Background color

Most passport/visa photos require a plain white or light gray background with no shadows, patterns, or other people visible. Visa photos for some countries (rare, but it happens) specify a different background color entirely — always check the specific country's current rule rather than assuming white is universal.

4. File size limit (for online applications)

Online application portals often cap the upload at a small file size — commonly under 100–300KB — separate from the pixel/DPI requirements. This is the part people get tripped up on: a photo can be the exact right pixel dimensions and still get rejected purely for being a 4MB file when the portal wants under 200KB.
